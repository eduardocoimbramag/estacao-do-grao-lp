# Documentação: Correções da Seção "Regiões Atendidas" (LAYDESK2) - Primeira Rodada

## Objetivo
Eliminar overflow na seção "Regiões Atendidas" (Section 4) no layout `laydesk2`, garantindo que todo o conteúdo caiba em "1 tela - menu" sem overflow vertical ou horizontal, preservando o esquema visual atual através de reduções sistemáticas de espaçamentos e tamanhos.

---

## Problemas Identificados (Primeira Rodada)

1. **OVERFLOW VERTICAL** - A seção está ultrapassando os limites de altura disponível (1 tela - menu = `calc(100vh - 4rem)`), causando overflow vertical.

2. **Espaçamentos excessivos** - Padding, margins e gaps muito grandes estão fazendo o conteúdo exceder o espaço disponível.

3. **Tamanhos de elementos grandes** - Título, mapa, cards e botões de navegação ocupam muito espaço vertical, necessitando reduções proporcionais.

---

## Análise do Problema

**Situação Atual:** A seção "Regiões Atendidas" tem uma estrutura de 2 colunas:
- **Coluna Esquerda:** Título + Mapa + 3 Cards de texto
- **Coluna Direita:** 2 Botões de navegação (Galeria e Blog)

**Causa do Overflow:** 
- Padding do container ainda muito generoso
- Gap do grid entre colunas pode ser reduzido
- Gap da coluna esquerda entre elementos pode ser reduzido
- Título com fonte grande e margin-bottom excessivo
- Mapa ocupando muito espaço (max-width 49.5% ainda pode ser reduzido)
- Cards com padding e margin-bottom que podem ser reduzidos
- Botões de navegação com padding e max-height que podem ser otimizados
- Falta de `overflow: hidden` na seção principal

**Nota:** Há CSS duplicado/conflictivo (linhas 734-858 e 1571-1660). Esta documentação propõe valores otimizados que devem substituir/atualizar ambos os blocos.

## Plano de Ajustes

### 1. **CRÍTICO: Garantir overflow: hidden na seção**

**Problema:** A seção não tem `overflow: hidden` aplicado (primeira seção não tem).

**Solução:**
- Adicionar `overflow: hidden` na seção principal
- Garantir `max-height: calc(100vh - 4rem)`

**CSS:**
```css
/* 1. Section: Altura viewport menos menu + Anti-overflow (LAYDESK2)
   Impacto: garante que a seção não ultrapasse os limites da tela */
.laydesk2-audience-section {
  height: calc(100vh - 4rem) !important;
  min-height: calc(100vh - 4rem) !important;
  max-height: calc(100vh - 4rem) !important;
  padding-top: 0 !important;
  padding-bottom: 0 !important;
  overflow: hidden !important; /* FORÇAR - impede overflow vertical/horizontal */
  overflow-x: hidden !important;
}
```

---

### 2. **Reduzir Padding do Container**

**Problema:** Padding do container ainda muito generoso (0.75rem - 1rem).

**Solução:** Reduzir padding vertical e horizontal para economizar espaço.

**CSS:**
```css
/* 2. Container interno: Padding reduzido + Flex shrink habilitado (LAYDESK2)
   Impacto: economiza espaço vertical e horizontal */
.laydesk2-audience-container {
  padding-top: 0.5rem !important; /* REDUZIDO de 0.75rem */
  padding-bottom: 0.5rem !important; /* REDUZIDO de 0.75rem */
  padding-left: 1rem !important; /* MANTIDO ou reduzido se necessário */
  padding-right: 1rem !important; /* MANTIDO ou reduzido se necessário */
  height: 100% !important;
  display: flex !important;
  flex-direction: column !important;
  justify-content: center !important;
  min-height: 0 !important; /* permite shrink */
  overflow: hidden !important; /* impede overflow interno */
}
```

## Notas Importantes

1. **PRIORIDADE MÁXIMA**: A correção do `overflow: hidden` na seção principal é **CRÍTICA** e deve ser aplicada primeiro. Sem isso, qualquer redução de espaçamento não resolverá o problema de overflow.

2. **Apenas LAYDESK2**: Todas as mudanças são exclusivas para o layout `laydesk2` (media query `1024px - 899px`). Não alterar outros layouts.

3. **CSS Duplicado**: Há CSS duplicado/conflictivo para esta seção. Após implementar as mudanças, verificar se há conflitos e consolidar/remover duplicações.

4. **Teste Visual**: Após aplicar as mudanças, testar em diferentes resoluções dentro do range `laydesk2` para garantir que:
   - Não há overflow vertical
   - Não há overflow horizontal
   - O conteúdo ainda está legível
   - O layout ainda está visualmente agradável
   - O conteúdo cabe em "1 tela - menu"

5. **Balanceamento**: As reduções propostas são conservadoras mas significativas. Se ainda houver overflow, considere as "Notas de Ajuste Fino" abaixo.

6. **Preservar Esquema Visual**: As reduções foram calculadas para manter proporções e hierarquia visual, apenas compactando o layout.
