# Documentação: Correções da Seção "Nossos Serviços" (LAYDESK2) - Quinta Rodada

## Objetivo
Corrigir problemas restantes na seção "Nossos Serviços" (Section 3) no layout `laydesk2`:
1. **CRÍTICO**: Corrigir corte superior e inferior do card central do carrossel (card em relevância/destaque)
2. Aproximar o subtítulo do título, reduzindo em 1/3 a distância entre eles

---

## Problemas Identificados (Quinta Rodada)

1. **CORTE DO CARD CENTRAL** - O card central do carrossel (ativo, em relevância) está com as partes superior e inferior cortadas. O card ativo tem `scale-105` (5% maior) mas o container do carrossel tem `overflow: hidden` e padding insuficiente, causando o corte.

2. **Subtítulo ainda distante do título** - Precisa reduzir a distância em 1/3 em relação ao valor atual.

---

## Análise do Problema Crítico

**Situação Atual:** O card ativo (central) tem `scale-105` aplicado via Tailwind (`scale-105` = transform: scale(1.05)), o que aumenta o card em 5%. Porém, o container do carrossel (`.laydesk2-servicos-carousel-container`) tem:
- `overflow: hidden !important` (para evitar overflow)
- `padding-top: 0.25rem !important` (muito reduzido)
- `padding-bottom: 0.375rem !important` (muito reduzido)

**Causa:** Quando o card é escalado para 105%, ele aumenta 5% em todas as direções (superior, inferior, esquerda, direita). Com padding vertical muito reduzido (0.25rem top, 0.375rem bottom), não há espaço suficiente para acomodar o aumento de 5% do card, resultando no corte das partes superior e inferior.

**Solução:** Aumentar o `padding-top` e `padding-bottom` do container do carrossel para dar espaço suficiente para o scale de 105%. Como o card tem escala de 105%, precisamos de pelo menos 2.5% de padding em cada lado (mas vamos usar valores mais seguros como 1rem ou 0.75rem).

**Cálculo Estimado:**
- Se um card tem altura de ~400px (exemplo), com `scale-105` ele fica ~420px (aumento de 20px = 10px top + 10px bottom)
- Para acomodar 10px de aumento, precisamos de pelo menos 10px (0.625rem) de padding em cada lado
- Vamos usar `padding-top: 0.75rem` e `padding-bottom: 0.75rem` para ter margem de segurança

## Plano de Ajustes

### 1. **CORREÇÃO CRÍTICA: Aumentar Padding Vertical do Container do Carrossel**

**Problema:** O card central (ativo) está sendo cortado nas partes superior e inferior devido ao `scale-105` combinado com padding vertical insuficiente.

**Solução:**
- Aumentar `padding-top` e `padding-bottom` do container do carrossel para dar espaço ao scale de 105%
- Valores recomendados: `padding-top: 0.75rem` e `padding-bottom: 0.75rem` (valores seguros que acomodam o scale)

**CSS:**
```css
/* 6. Container do carrossel: Padding aumentado para acomodar scale-105 do card ativo (LAYDESK2)
   Impacto: corrige o corte superior/inferior do card central (ativo) */
.laydesk2-servicos-carousel-container {
  padding-top: 0.75rem !important; /* AUMENTADO de 0.25rem para acomodar scale-105 */
  padding-bottom: 0.75rem !important; /* AUMENTADO de 0.375rem para acomodar scale-105 */
  flex: 1 !important;
  min-height: 0 !important;
  display: flex !important;
  align-items: center !important;
  max-height: 100% !important;
  overflow: hidden !important; /* mantém para evitar overflow horizontal */
  overflow-x: hidden !important;
}
```

**Nota:** Se após testar `0.75rem` ainda houver corte, aumentar para `1rem` ou `1.25rem`. Mas `0.75rem` deve ser suficiente para acomodar o scale de 105%.

---

### 2. **Reduzir Distância Título→Subtítulo em 1/3**

**Problema:** A distância entre título e subtítulo precisa ser reduzida em 1/3 em relação ao valor atual.

**Análise:** Atualmente o `margin-top` do subtítulo está em `-0.1875rem`. Para reduzir a distância em 1/3, precisamos tornar o margin mais negativo.

**Cálculo:**
- Distância atual: `-0.1875rem`
- Para reduzir em 1/3, tornar mais negativo: `-0.1875rem * (4/3) = -0.25rem`
- Ou mais conservador: aumentar para `-0.25rem` (aproximadamente 33% mais negativo)

**Decisão:** Vamos usar `-0.25rem` (aumento de ~33% no valor negativo).

**CSS:**
```css
/* 5. Parágrafo: Margin top negativo aumentado (reduz distância em 1/3) (LAYDESK2)
   Impacto: reduz a distância entre título e subtítulo em 1/3 */
.laydesk2-servicos-description {
  font-size: 0.9375rem !important;
  line-height: 1.375rem !important;
  margin-top: -0.25rem !important; /* AUMENTADO de -0.1875rem (redução de 1/3) */
}
```

**Alternativa (se -0.25rem não for suficiente):**
```css
.laydesk2-servicos-description {
  margin-top: -0.3125rem !important; /* AUMENTADO de -0.1875rem (redução mais agressiva) */
}
```

## Notas Importantes

1. **PRIORIDADE MÁXIMA**: A correção do padding do container do carrossel é **CRÍTICA** e deve ser aplicada primeiro. O card ativo (central) com `scale-105` precisa de espaço vertical suficiente para não ser cortado.

2. **Apenas LAYDESK2**: Todas as mudanças são exclusivas para o layout `laydesk2` (media query `1024px - 899px`). Não alterar outros layouts.

3. **Padding do Carrossel - Cálculo**: O card ativo tem `scale-105` (5% maior). Para um card de ~400px de altura, isso significa ~20px de aumento (10px top + 10px bottom). Com `padding-top: 0.75rem` (12px) e `padding-bottom: 0.75rem` (12px), temos espaço suficiente para acomodar o scale.

4. **Se 0.75rem Ainda Não Funcionar**: Se após aplicar `0.75rem` ainda houver corte em alguns cards, aumentar para:
   - `1rem` (16px - mais seguro)
   - `1.25rem` (20px - garantia total)

5. **Redução de Distância**: O cálculo de redução de 1/3 resulta em `-0.25rem`. Se isso não for visualmente suficiente, podemos aumentar para `-0.3125rem` (mais agressivo).

6. **Teste Visual**: Após aplicar as mudanças, testar em diferentes resoluções dentro do range `laydesk2` para garantir que:
   - O card central (ativo) não está mais cortado nas partes superior e inferior
   - Não há overflow vertical
   - O subtítulo está mais próximo do título
   - O conteúdo ainda cabe em "1 tela - menu"

7. **Balanceamento**: Aumentar o padding do carrossel pode reduzir ligeiramente o espaço disponível para os cards. Se necessário, podemos reduzir outros espaçamentos (mas apenas se realmente necessário).
