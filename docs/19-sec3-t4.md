## Análise do Problema Crítico

**Situação Atual:** Os cards estão configurados com `min-width: 32%` e `max-width: 32%`, mas ainda assim 4 cards estão aparecendo.

**Causa:** Em uma tela de laydesk2 (1024px-1279px), com `32%` de largura por card, ainda cabem 4 cards. O cálculo:
- Largura da tela: ~1200px (exemplo)
- Largura por card (32%): ~384px
- Cards cabíveis: 1200px / 384px = 3.125 cards → **4 cards visíveis**

**Solução Definitiva:** Precisamos aumentar a largura mínima dos cards para `33.333%` (valor original do Tailwind) ou próximo disso. Com `33.333%`:
- Largura por card (33.333%): ~400px
- Cards cabíveis: 1200px / 400px = 3 cards → **3 cards visíveis** ✅

**Alternativa:** Se `33.333%` ainda permitir 4 cards em algumas resoluções, usar `34%` ou `35%` para garantir que apenas 3 cabem.

**Decisão:** Vamos usar `33.333%` primeiro (valor original do Tailwind que funcionava), garantindo que apenas 3 cards sejam visíveis.

## Plano de Ajustes

### 1. **CORREÇÃO CRÍTICA DEFINITIVA: Ajustar Largura dos Cards para 33.333%**

**Problema:** Cards com `32%` de largura ainda causam 4 cards visíveis.

**Solução:**
- Aumentar `min-width` e `max-width` dos cards para `33.333%` (valor original do Tailwind)
- `33.333%` garante matematicamente que apenas 3 cards cabem na tela (100% / 33.333% = 3 cards exatos)
- Este é o valor que o Tailwind usa (`lg:min-w-[33.333%]`) e que funcionava antes

**CSS:**
```css
/* [SEC3] Cards - Largura ajustada para 33.333% (valor original Tailwind) (LAYDESK2)
   Impacto: garante matematicamente que apenas 3 cards sejam visíveis (100% / 33.333% = 3) */
.laydesk2-servicos-card {
  min-width: 33.333% !important; /* AJUSTADO de 32% para 33.333% (valor original Tailwind) */
  max-width: 33.333% !important; /* limita a largura máxima */
}
```

**Nota:** Se após testar `33.333%` ainda aparecerem 4 cards, aumentar para `34%` ou `35%`. Mas `33.333%` deve funcionar pois é o valor matemático exato para 3 cards.

### 2. **Reduzir Distância Título→Subtítulo em 1/3**

**Problema:** A distância entre título e subtítulo precisa ser reduzida em 1/3.

**Análise:** Atualmente o `margin-top` do subtítulo está em `-0.125rem`. Para reduzir a distância em 1/3, precisamos torná-lo mais negativo.

**Cálculo:**
- Distância atual: `-0.125rem`
- Redução de 1/3: tornar a distância 2/3 do atual
- Nova distância: `-0.125rem * (2/3) = -0.0833rem` ≈ `-0.1875rem` (mais próximo)

**Alternativa mais simples:** Aumentar o valor negativo para `-0.25rem` (dobrar o valor atual), o que reduzirá a distância ainda mais.

**Decisão:** Vamos usar `-0.1875rem` (redução matemática exata de 1/3) ou `-0.25rem` (mais seguro e visível).

**CSS:**
```css
/* [SEC3] Subtítulo - Margin top negativo aumentado (reduzir distância em 1/3) (LAYDESK2)
   Impacto: reduz a distância entre título e subtítulo em 1/3 */
.laydesk2-servicos-description {
  margin-top: -0.1875rem !important; /* AUMENTADO de -0.125rem (redução de 1/3) */
}
```

**Alternativa (se -0.1875rem não for suficiente):**
```css
.laydesk2-servicos-description {
  margin-top: -0.25rem !important; /* AUMENTADO de -0.125rem (redução mais agressiva) */
}
```

6. **Teste Visual**: Após aplicar as mudanças, testar em diferentes resoluções dentro do range `laydesk2` para garantir que:
   - Apenas 3 cards são visíveis (1 central + 2 adjacentes)

7. **Validação Matemática**: `33.333%` é o valor correto porque:

   - Este é o valor usado pelo Tailwind originalmente
