# Documentação: Correções Críticas da Seção "Nossos Serviços" (LAYDESK2) - Terceira Rodada

## Objetivo
Corrigir problemas críticos na seção "Nossos Serviços" (Section 3) no layout `laydesk2`:
1. **CRÍTICO**: Corrigir overflow horizontal - garantir que apenas 3 cards sejam visíveis (1 central + 2 adjacentes)
2. Eliminar overflow vertical no carrossel
3. Reduzir padding-bottom dos cards
4. Colar ainda mais o subtítulo ao título
5. Reduzir sutilmente os cards para caber tudo sem overflow

---

## Problemas Identificados (Terceira Rodada)

1. **OVERFLOW HORIZONTAL CRÍTICO** - Aparecem 4 cards quando deveriam aparecer apenas 3 (1 central + 2 adjacentes)
2. **Overflow vertical ainda presente** - O conteúdo do carrossel ainda ultrapassa os limites da seção
3. **Padding-bottom dos cards muito grande** - Espaço excessivo na parte inferior dos cards
4. **Subtítulo ainda distante do título** - Precisa colar mais (atualmente `margin-top: 0` mas pode precisar de ajuste de line-height)
5. **Cards podem ser reduzidos sutilmente** - Sem causar o problema de 4 cards aparecerem

---

## Análise do Problema Crítico

**Causa Raiz:** Ao reduzir a largura dos cards para `28%` no CSS, os cards ficaram menores e agora cabem 4 cards na tela em vez de 3. O carrossel Embla usa `lg:min-w-[33.333%]` no Tailwind, que garante que apenas 3 cards sejam visíveis. Ao sobrescrever para `28%`, quebramos esse comportamento.

**Solução:** Precisamos aumentar a largura mínima dos cards para ~32-33% para garantir que apenas 3 cards sejam visíveis, e depois compensar reduzindo outros elementos (padding-bottom, espaçamentos) para eliminar o overflow vertical.

## Plano de Ajustes

### 1. **CORREÇÃO CRÍTICA: Ajustar Largura dos Cards para 3 Visíveis**

**Problema:** Cards com `28%` de largura causam 4 cards visíveis.

**Solução:**
- Aumentar `min-width` e `max-width` dos cards para `32%` (entre 28% e 33.333%)
- Isso garante que apenas 3 cards sejam visíveis (1 central + 2 adjacentes)
- `32%` é um bom compromisso: permite 3 cards visíveis mas ainda é menor que os 33.333% originais

**CSS:**
```css
/* [SEC3] Cards - Largura ajustada para 3 cards visíveis (LAYDESK2)
   Impacto: garante que apenas 3 cards sejam visíveis (1 central + 2 adjacentes) */
.laydesk2-servicos-card {
  min-width: 32% !important; /* AJUSTADO de 28% para garantir 3 cards visíveis */
  max-width: 32% !important; /* limita a largura máxima */
}
```

---

### 2. **Garantir Overflow Horizontal Oculto**

**Problema:** Precisamos garantir que qualquer overflow horizontal seja oculto.

**Solução:**
- Garantir que o container do carrossel tenha `overflow-x: hidden`
- Garantir que a seção tenha `overflow-x: hidden`

**CSS:**
```css
/* [SEC3] Seção - Overflow horizontal oculto (LAYDESK2)
   Impacto: garante que overflow horizontal seja oculto */
.laydesk2-servicos-section {
  overflow-x: hidden !important; /* FORÇAR - impede overflow horizontal */
}

/* [SEC3] Container do carrossel - Overflow horizontal oculto (LAYDESK2)
   Impacto: garante que overflow horizontal do carrossel seja oculto */
.laydesk2-servicos-carousel-container {
  overflow-x: hidden !important; /* FORÇAR - já existe overflow: hidden, mas garantir */
}
```

---

### 3. **Reduzir Padding-Bottom dos Cards**

**Problema:** Padding-bottom dos cards muito grande (`0.875rem`).

**Solução:**
- Reduzir `padding-bottom` do conteúdo do card para `0.625rem`

**CSS:**
```css
/* [SEC3] Conteúdo do card - Padding bottom reduzido (LAYDESK2)
   Impacto: reduz espaço inferior do card */
.laydesk2-servicos-card-content {
  padding-bottom: 0.625rem !important; /* REDUZIDO de 0.875rem */
}
```

---

### 4. **Colar Ainda Mais o Subtítulo ao Título**

**Problema:** Subtítulo ainda parece distante do título mesmo com `margin-top: 0`.

**Solução:**
- O `margin-top: 0` já está aplicado, mas podemos ajustar o `line-height` do título para reduzir espaço abaixo
- Ou usar `margin-bottom` negativo no título
- Alternativa: ajustar `margin-top` negativo no subtítulo (ex: `-0.125rem`)

**Análise:** Como o `margin-top: 0` já está aplicado no `* + *`, o espaço pode estar vindo do `line-height` do título. Vamos usar `margin-top` negativo no subtítulo para "puxá-lo" para cima.

**CSS:**
```css
/* [SEC3] Subtítulo - Margin top negativo para colar ao título (LAYDESK2)
   Impacto: cola o subtítulo ao título usando margin negativo */
.laydesk2-servicos-description {
  margin-top: -0.125rem !important; /* NEGATIVO - puxa o subtítulo para cima */
}
```

---

### 5. **Reduzir Sutilmente Outros Elementos para Compensar**

**Problema:** Com cards em `32%` (maior que `28%`), pode ser necessário reduzir outros elementos para compensar e eliminar overflow vertical.

**Solução:**
- Reduzir ainda mais o `padding-bottom` do container (de `0.75rem` para `0.5rem`)
- Reduzir ainda mais o `padding-bottom` do carrossel container (de `0.5rem` para `0.375rem`)

**CSS:**
```css
/* [SEC3] Container - Padding bottom ainda mais reduzido (LAYDESK2)
   Impacto: reduz espaço inferior, compensando largura maior dos cards */
.laydesk2-servicos-container {
  padding-bottom: 0.5rem !important; /* REDUZIDO de 0.75rem */
}

/* [SEC3] Container do carrossel - Padding bottom ainda mais reduzido (LAYDESK2)
   Impacto: reduz espaço inferior do carrossel */
.laydesk2-servicos-carousel-container {
  padding-bottom: 0.375rem !important; /* REDUZIDO de 0.5rem */
}
```

## Notas Importantes

1. **PRIORIDADE MÁXIMA**: A correção da largura dos cards (28% → 32%) é **CRÍTICA** e deve ser aplicada primeiro. Isso resolve o problema de 4 cards aparecendo.

2. **Apenas LAYDESK2**: Todas as mudanças são exclusivas para o layout `laydesk2` (media query `1024px - 899px`). Não alterar outros layouts.

3. **Largura dos Cards**: `32%` é um compromisso entre os `33.333%` originais (que garantiam 3 cards) e os `28%` anteriores (que causavam 4 cards). Com `32%`, ainda temos cards ligeiramente menores que os originais, mas mantemos o comportamento correto de 3 cards visíveis.

4. **Overflow Horizontal**: O `overflow-x: hidden` garante que qualquer conteúdo que ultrapasse horizontalmente seja oculto, mesmo que os cards tenham largura correta.

5. **Margin Negativo**: O `margin-top: -0.125rem` no subtítulo pode parecer pequeno, mas é suficiente para "colar" visualmente o subtítulo ao título, removendo qualquer espaço aparente.

6. **Compensação**: Ao aumentar a largura dos cards para `32%`, aumentamos ligeiramente o espaço ocupado. Para compensar, reduzimos os `padding-bottom` do container e do carrossel, mantendo o conteúdo dentro de "1 tela - menu".

7. **Teste Visual**: Após aplicar as mudanças, testar em diferentes resoluções dentro do range `laydesk2` para garantir que:
   - Apenas 3 cards são visíveis (1 central + 2 adjacentes)
   - Não há overflow horizontal
   - Não há overflow vertical
   - Tudo cabe em "1 tela - menu"
