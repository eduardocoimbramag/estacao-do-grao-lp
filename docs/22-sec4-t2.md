# Documentação: Ajustes Visuais da Seção "Regiões Atendidas" (LAYDESK2) - Segunda Rodada

## Objetivo
Ajustar elementos visuais da seção "Regiões Atendidas" (Section 4) no layout `laydesk2`:
1. Aumentar padding dos botões da direita (Galeria e Blog) para melhor alinhamento e simetria visual
2. Aumentar tamanho da fonte dos cards da esquerda (3 cards com informações sobre regiões) para melhor legibilidade

---

## Problemas Identificados (Segunda Rodada)

1. **Padding dos botões da direita insuficiente** - Os dois botões de navegação (Galeria de experiências e Blog) na coluna direita têm padding vertical muito reduzido (`0.5rem`), fazendo com que não aproveitem bem o espaço disponível e não fiquem visualmente alinhados com o conteúdo da coluna esquerda.

2. **Fonte dos cards da esquerda muito pequena** - O texto dos 3 cards informativos (Recife e Região Metropolitana, Pernambuco/Paraíba/Alagoas, Todo o Brasil) está com `font-size: 0.8125rem` (13px), que é muito pequeno para o espaço disponível, dificultando a legibilidade.

---

## Análise do Problema

**Situação Atual:** 
- **Botões da direita:** `padding-top: 0.5rem`, `padding-bottom: 0.5rem` - valores muito reduzidos que foram aplicados na primeira rodada para eliminar overflow, mas agora que o layout está estável, podem ser aumentados para melhor aproveitamento do espaço e alinhamento visual.
- **Cards da esquerda:** `font-size: 0.8125rem` (13px), `line-height: 1.0625rem` - valores muito pequenos que foram reduzidos na primeira rodada, mas agora podem ser aumentados para melhor legibilidade sem comprometer o layout.

**Objetivo dos Ajustes:**
- Aumentar padding dos botões para melhor alinhamento vertical com o conteúdo da esquerda
- Aumentar tamanho da fonte dos cards para melhor legibilidade, aproveitando o espaço disponível
- Manter o layout sem overflow (garantir que ainda cabe em "1 tela - menu")

---

## Estado Atual do CSS

### Valores Atuais (após primeira rodada - 21-sec4-t1):

**Botões de navegação (coluna direita):**
- `padding-top: 0.5rem` (8px)
- `padding-bottom: 0.5rem` (8px)
- `max-height: 13rem` (208px)

**Texto dos cards (coluna esquerda):**
- `font-size: 0.8125rem` (13px)
- `line-height: 1.0625rem` (17px)

---

## Plano de Ajustes

### 1. **Aumentar Padding dos Botões da Direita**

**Problema:** Padding vertical muito reduzido (`0.5rem`) não aproveita bem o espaço disponível e não alinha visualmente com o conteúdo da esquerda.

**Solução:** Aumentar padding vertical dos botões para melhor aproveitamento do espaço e alinhamento visual. Baseado na referência visual (linha vermelha indicativa), podemos aumentar para `0.75rem` ou `1rem` mantendo proporção.

**Recomendação:** Usar `0.75rem` como valor intermediário (aumento de 50%), que oferece melhor visual sem risco de overflow. Se necessário e houver espaço suficiente, pode-se aumentar para `1rem`.

**CSS:**
```css
/* 9. Botões de navegação: Padding aumentado para melhor alinhamento (LAYDESK2)
   Impacto: melhora aproveitamento do espaço e alinhamento visual com coluna esquerda */
.laydesk2-audience-nav-button {
  padding-top: 0.75rem !important; /* AUMENTADO de 0.5rem (aumento de 50%) */
  padding-bottom: 0.75rem !important; /* AUMENTADO de 0.5rem (aumento de 50%) */
  min-height: 0 !important;
  max-height: 13rem !important; /* MANTIDO - não alterar altura máxima para evitar overflow */
  flex: 1 1 auto !important;
  height: auto !important;
  overflow: hidden !important;
}
```

**Alternativa (se 0.75rem não for suficiente):**
```css
padding-top: 1rem !important; /* AUMENTADO de 0.5rem (aumento de 100%) */
padding-bottom: 1rem !important; /* AUMENTADO de 0.5rem (aumento de 100%) */
```

**Nota:** O `max-height: 13rem` deve ser mantido para evitar overflow. O aumento do padding interno não altera a altura máxima dos botões, apenas o espaçamento interno do conteúdo.

---

### 2. **Aumentar Tamanho da Fonte dos Cards da Esquerda**

**Problema:** Fonte muito pequena (`0.8125rem` = 13px) dificulta a legibilidade e não aproveita bem o espaço disponível nos cards.

**Solução:** Aumentar `font-size` e `line-height` proporcionalmente para melhor legibilidade.

**Recomendação:** 
- `font-size: 0.875rem` (14px) - aumento de ~8% (de 13px para 14px)
- `line-height: 1.125rem` (18px) - proporcional ao font-size, mantendo ratio de ~1.29

Este aumento é moderado mas perceptível, melhorando a legibilidade sem comprometer o layout.

**CSS:**
```css
/* Texto dos cards: Tamanho aumentado para melhor legibilidade (LAYDESK2)
   Impacto: melhora legibilidade do texto nos 3 cards informativos */
.laydesk2-audience-card-text {
  font-size: 0.875rem !important; /* AUMENTADO de 0.8125rem (13px → 14px) */
  line-height: 1.125rem !important; /* AUMENTADO de 1.0625rem (17px → 18px) */
}

/* Texto em negrito dentro dos cards: Herdar tamanho (mantido) */
.laydesk2-audience-card-text strong {
  font-size: inherit !important;
  font-weight: bold !important;
}

/* Cards - Texto "100 doses" e "600 doses": Herdar tamanho (mantido) */
.laydesk2-audience-card-text strong.text-base,
.laydesk2-audience-card-text strong.sm\:text-lg {
  font-size: inherit !important;
  font-weight: bold !important;
}
```

**Alternativa (se 0.875rem ainda for pequeno):**
```css
font-size: 0.9375rem !important; /* AUMENTADO de 0.8125rem (13px → 15px) - aumento mais significativo */
line-height: 1.25rem !important; /* AUMENTADO de 1.0625rem (17px → 20px) - proporcional */
```

## Notas Importantes

1. **Apenas LAYDESK2**: Todas as mudanças são exclusivas para o layout `laydesk2` (media query `1024px - 899px`). Não alterar outros layouts.

2. **Manter max-height**: O `max-height: 13rem` dos botões deve ser mantido para evitar overflow. O aumento do padding interno não altera a altura máxima total dos botões, apenas o espaçamento interno.

3. **Teste Visual**: Após aplicar as mudanças, testar em diferentes resoluções dentro do range `laydesk2` para garantir que:
   - Os botões ficam melhor alinhados visualmente com o conteúdo da esquerda
   - O texto dos cards está mais legível
   - Não há overflow vertical ou horizontal
   - O conteúdo ainda cabe em "1 tela - menu"

4. **Ajustes Progressivos**: 
   - Para botões: Começar com `0.75rem` e, se houver espaço suficiente e necessidade, aumentar para `1rem`
   - Para texto: Começar com `0.875rem` e, se ainda estiver pequeno, aumentar para `0.9375rem`

5. **Balanceamento Visual**: Os aumentos propostos são moderados para manter o equilíbrio visual do layout, melhorando a usabilidade sem comprometer o design compacto.
